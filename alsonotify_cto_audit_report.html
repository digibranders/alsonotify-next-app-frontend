1. Executive Summary

Platform Overview

Alsonotify is a SaaS project management and invoicing platform built for teams and agencies. It provides workspace management, task tracking with time logging, employee management, invoicing with PDF generation, calendar integration (Microsoft Graph), email integration, leave management, and real-time notifications via WebSocket.


The platform follows a monorepo structure with separate codebases for the main frontend application, an admin dashboard, a landing page, and the backend API server.

Architecture Summary

Frontend

Next.js 16.1.6 with React 19, TypeScript 5.9 (strict), Tailwind CSS 4.1, Ant Design 6.1, TanStack React Query v5

Backend

Fastify v5.3.2 with Node.js 22, Prisma v6.18, PostgreSQL 15, WebSocket (ws), node-cron

Infrastructure

DigitalOcean droplet (backend), PM2 process manager, S3 file storage, Microsoft Graph API

CI/CD

GitHub Actions — backend auto-deploys on push to main, frontend CI-only (lint/typecheck/test/build)

Launch Readiness Overview

The platform demonstrates solid engineering fundamentals — TypeScript strict mode, DOMPurify sanitization, CSP headers, Sentry error tracking, React Query caching, and structured Prisma ORM usage. However, critical security vulnerabilities (exposed API keys, JWT algorithm bypass, missing auth guards) and operational gaps (no token revocation, minimal test coverage, missing database indexes) make it not ready for public beta in its current state.

6. Critical Issues — Full Detail

ID Severity Issue Component Explanation Recommended Fix
C-1 Critical Exposed production secrets in .env Backend .env Active API keys for OpenAI, Anthropic, AWS (access key + secret), Azure AD, Google OAuth, Brevo email service, JWT signing secret, and super admin password stored in plaintext. If this repository is or was ever pushed to a shared/public remote, all credentials are compromised. 1. Immediately rotate ALL exposed keys and passwords.
2. Remove .env from version control history (git filter-repo).
3. Add .env to .gitignore (already present but files exist).
4. Use GitHub Secrets exclusively for CI/CD (already partially done).
5. Audit AWS CloudTrail for unauthorized usage.
C-2 Critical JWT algorithm not enforced Backend utils/jwt.ts The jsonwebtoken.verify() call does not specify the expected algorithm. The alg: "none" attack allows an attacker to create a token with no cryptographic signature that passes verification. Add algorithms: ['HS256'] to all jwt.verify() calls:
jwt.verify(token, secret, { algorithms: ['HS256'] })
C-3 Critical S3 upload not verified on confirm Backend file.service.ts After presigned URL upload, client calls /confirm. Backend creates a FileAttachment record without calling s3.headObject() to verify the object exists. Allows phantom records. Before saving metadata, verify S3 object:
await s3.headObject({ Bucket, Key })
Return 404 if object missing.
C-4 Critical Developer emails in NEXT_PUBLIC Frontend .env + roleUtils.ts NEXT_PUBLIC_DEVELOPER_EMAILS is shipped to the browser bundle. Used for client-side "developer" role detection. Any user can inspect the bundle to see developer accounts and potentially spoof roles. Move developer role assignment to server-side. Backend should set isDeveloper flag in JWT payload based on server-side config. Remove NEXT_PUBLIC_DEVELOPER_EMAILS entirely.
C-5 Critical Turnstile CAPTCHA test keys Frontend + Backend .env Both site key (1x00000000000000000000AA) and secret key are Cloudflare test values that always pass. Zero bot protection on auth endpoints. Replace with production Turnstile keys from Cloudflare dashboard. Ensure TURNSTILE_SECRET_KEY is server-side only.
C-6 Critical $queryRawUnsafe in production Backend services Used in invoice.service.ts and workspace.service.ts. Currently parameterized but $queryRawUnsafe doesn't enforce this at the type level. Future string interpolation = SQL injection. Migrate to $queryRaw tagged template literals which enforce parameterization at the TypeScript level.
C-7 Critical ADMIN_JWT_SECRET not startup-validated Backend config/env.ts Zod env schema validates JWT_SECRET_KEY but ADMIN_JWT_SECRET is entirely absent. Server starts without it, admin routes return 500 at runtime. Add ADMIN_JWT_SECRET: z.string().min(32) to the Zod env validation schema.
H-1 High Missing DB indexes on FK columns Prisma schema Notification, LeaveRequest, WorkspaceRequirement, OAuthToken models lack @@index on frequently-queried foreign keys (user_id, company_id). Dashboard queries perform full table scans. Add @@index([user_id]) and @@index([company_id]) to affected models. Create migration and deploy.
H-2 High CORS defaults to localhost Backend fastify.ts If ALLOWED_ORIGINS is not set in production, CORS origin defaults to http://localhost:3000,http://localhost:3001. No startup validation catches this. Add env validation: throw startup error if NODE_ENV=production and ALLOWED_ORIGINS is empty.
H-3 High RBAC not scoped to company Backend permissions.ts checkPermission() filters by role_id only. Does not validate the role belongs to user's company. If role IDs collide across companies, cross-tenant access is possible. Add company filter: where: { role: { company_id: user.company_id } }
H-4 High No file type whitelist Backend file.service.ts Upload accepts any MIME type. Executables, scripts, and malware can be stored in S3 under company paths. Add allowlist: ['image/jpeg','image/png','application/pdf','text/csv',...]. Reject unknown types with 415.
H-5 High Unsanitized email HTML Frontend MailPage.tsx Email HTML body rendered via dangerouslySetInnerHTML without DOMPurify. Notes use sanitization, but mail does not. Apply sanitizeEmailHtml() (already exists in codebase) to mail HTML before rendering.
H-6 High JWT not in HttpOnly cookie Frontend auth JWT stored in JS-accessible cookie. Any XSS vulnerability (e.g., H-5) can exfiltrate auth tokens. Set cookie with httpOnly: true flag. Requires backend to set cookie on login response instead of returning token in body.
H-7 High No server-side auth middleware Frontend No Next.js middleware.ts. Auth check is client-side only. Protected pages briefly flash before redirect. Create src/middleware.ts that validates JWT cookie on protected routes and redirects 302 to /login.
H-8 High No token revocation Backend auth Logout removes client cookie but JWT remains valid server-side until expiry. Password change doesn't invalidate old tokens. Implement Redis-backed token blacklist. On logout/password change, add token jti to blacklist with TTL matching token expiry.
H-9 High No @fastify/helmet on backend Backend fastify.ts Backend API responses lack HSTS, X-Frame-Options, X-Content-Type-Options, and CSP headers. Helmet is never imported or registered. Frontend has these via Next.js, but direct API access is unprotected. Install and register @fastify/helmet with appropriate configuration for API responses.
H-10 High Admin routes lack global auth guard Backend routes.ts Private routes have a global preHandler for auth, but admin routes require each sub-file to individually add verifySuperAdminToken. A missing guard = public admin endpoint. Add global preHandler: [verifySuperAdminToken] to the admin route prefix registration in routes.ts.
H-11 High No CSRF protection for admin Backend admin auth Admin auth uses httpOnly cookie without CSRF tokens. CORS alone doesn't prevent cross-site form submissions from invoking admin actions. Add CSRF token validation to admin routes, or migrate admin auth to Bearer tokens (consistent with user auth).
H-12 High Cross-tenant comment access Prisma schema + Comment service Comment model has no company_id field. Comments linked via polymorphic comment_type_id. User knowing another company's task ID could read its comments. Add company_id to Comment model. Filter all comment queries by company_id = user.company_id.
H-13 High Login rate limit per-email only Backend auth-routes.ts Rate limit keyGenerator returns body?.email || req.ip. Attacker can brute-force unlimited email accounts from one IP without triggering IP-level limits. Apply compound rate limit: per-IP (50/min) AND per-email (10/min). Add IP rate limit as separate middleware.
