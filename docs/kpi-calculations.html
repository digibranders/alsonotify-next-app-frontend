Requirement Tab

KPI Cards GET /report/requirements

Total Requirements

Count of all non-deleted, non-archived requirements that overlap the selected date range. Excludes statuses: Draft, Waiting, Submitted, Rejected. Includes Completed (for KPI accuracy).

totalRequirements = COUNT(requirements)
// WHERE is_deleted = false AND is_archived = false
// AND status NOT IN (Draft, Waiting, Submitted, Rejected)
// AND start_date <= filterEndDate AND end_date >= filterStartDate

On Time Completed

Requirements completed within budget (engaged hours ≤ allotted hours). Over-budget completions are excluded and counted under "Delayed" instead.

onTimeCompleted = totalCompleted − delayedCompleted

// totalCompleted = requirements with status = "Completed"
// delayedCompleted = completed requirements WHERE engagedHrs > allottedHrs

In Progress

Requirements currently being worked on. Includes statuses: Assigned, In_Progress, Review, Revision, Delayed, On_Hold.

inProgress = COUNT(requirements WHERE status IN (Assigned, In_Progress, Review, Revision, Delayed, On_Hold))

Delayed

Requirements that are past their end date (not yet completed) + completed requirements that went over budget. The "+Xh" suffix shows total extra hours logged across all delayed-completed requirements.

delayed = COUNT(reqs WHERE status ≠ Completed AND end_date < NOW)

totalExtraHrs = SUM(engagedHrs − allottedHrs)
// For each completed requirement where engagedHrs > allottedHrs

Efficiency

Percentage of requirements completed on time and within budget, relative to total requirements.

efficiency = ROUND((onTimeCompleted / totalRequirements) × 100)%
// Returns 0 if totalRequirements = 0

Requirement Tab

Row Metrics

Hours Utilization

Per-requirement bar showing engaged (logged) hours vs. allotted (budget) hours. Color-coded by status.

allottedHrs = SUM(task.estimated_time) for all tasks in requirement
engagedHrs = SUM(worklog.time_in_seconds) / 3600 for all worklogs
remaining = allottedHrs − engagedHrs
fillPercent = MIN(engagedHrs / allottedHrs, 1) × 100
Normal — within budget Warning — ≥ 80% used Bleeding — over budget

Revenue / P&L

Revenue from quoted price (or budget fallback) and profit/loss for each requirement.

revenue = quoted_price || budget || 0
cost = SUM((worklog_seconds / 3600) × member_hourly_rate)
profit = revenue − cost

Tasks Tab

KPI Cards GET /report/tasks

Total Tasks

Count of active, non-deleted tasks (excluding sub-tasks unless they are revisions) within the date range.

totalTasks = COUNT(tasks WHERE is_active = true AND (parent_id IS NULL OR is_revision = true))
// AND start_date BETWEEN filterStart..filterEnd

On Time Completed

Tasks completed within their estimated hours (no extra hours logged).

onTimeCompleted = COUNT(tasks WHERE status = "Completed" AND engagedHrs ≤ allottedHrs)

In Progress

Tasks with active statuses (not past deadline).

inProgress = COUNT(tasks WHERE status IN (In_Progress, Assigned, Review))

Delayed

Tasks with explicit "Delayed" status OR past their due date (not yet completed). "+Xh" shows total extra hours from over-budget completions.

delayed = COUNT(tasks WHERE status = "Delayed" OR (end_date < NOW AND status ≠ "Completed"))

delayedCompleted = COUNT(tasks WHERE status = "Completed" AND engagedHrs > allottedHrs)
totalExtraHrs = SUM(engagedHrs − allottedHrs) for delayedCompleted tasks

Efficiency

Percentage of tasks completed on time (within estimated hours) relative to total tasks.

efficiency = ROUND((onTimeCompleted / totalTasks) × 100)%
// Returns 0 if totalTasks = 0

Tasks Tab

Row Metrics

Hours Variance

Per-task bar showing engaged vs. allotted hours with color-coded status.

allottedHrs = task.estimated_time
engagedHrs = SUM(worklog.time_in_seconds) / 3600
remaining = allottedHrs − engagedHrs
fillPercent = MIN(engagedHrs / allottedHrs, 1) × 100
Normal — within estimate Warning — ≥ 80% used Over — exceeded estimate

Employee Tab

KPI Cards GET /report/employees

Total Expenses

Aggregate labour cost = sum of each employee's logged hours × their hourly rate.

totalExpenses = SUM(employee.engagedHrs × employee.hourlyCost)
// Computed across ALL employees matching filters (not paginated)

Total Revenue

Revenue generated, prorated by each employee's contribution (logged hours) to requirements with a quoted price or budget.

totalRevenue = SUM(employee_revenue)

// Per employee, per worklog:
revenue += requirement_price × (worklog_seconds / total_requirement_seconds)
// requirement_price = quoted_price || budget || 0
// In-house requirements with no price = $0 revenue

Net Profit

Total Revenue minus Total Expenses. Positive = profitable workforce.

netProfit = totalRevenue − totalExpenses

Avg. Rate/Hr

Average hourly cost rate across all filtered employees.

avgRatePerHr = SUM(employee.hourlyCost) / COUNT(employees)

// hourlyCost per employee:
// 1. Use hourly_rates if set
// 2. Fallback: salary_yearly / annualWorkingHours
annualWorkingHours = (daysPerWeek × 52 − holidays − leaves) × dailyHours

Occupancy

Average workforce utilization: ratio of engaged (logged) hours to available capacity. ≥70% is considered healthy.

avgOccupancy = AVG(employee.utilization)

// Per employee:
utilization = (engagedHrs / periodCapacity) × 100

periodCapacity = workingDaysInRange × dailyHours
// workingDaysInRange = count of non-holiday weekdays matching the user's schedule
// within the selected date range (considers per-user working_hours config)
Healthy — ≥ 70% Low — < 70%

Efficiency

Average on-time task delivery rate across all employees. ≥75% is considered good.

avgEfficiency = AVG(employee.efficiency)

// Per employee:
efficiency = (completedOnTime / totalAssignedTasks) × 100

// "On time" = task.status = "Completed"
// AND (task.end_date IS NULL OR completed_at ≤ task.end_date)
// Tasks sourced from task_members junction table (per-user assignment)
// Returns 0 if employee has 0 assigned tasks
Good — ≥ 75% Low — < 75%

Employee Tab

Row Metrics

Tasks Performance

Per-employee task breakdown with colour-coded status dots.

assigned = total tasks assigned via task_members junction
● completed = tasks WHERE status = "Completed"
● inProgress = tasks WHERE status IN (In_Progress, Assigned, Review) AND NOT overdue
● delayed = tasks WHERE status = "Delayed" OR (end_date < NOW AND status ≠ "Completed")

Categories are mutually exclusive: overdue tasks count as "delayed" even if status is "In_Progress".

Load (Occupancy)

Per-employee utilization bar. Red bar = logged hours exceed capacity (>100%).

load = ROUND((engagedHrs / periodCapacity) × 100)%

Expenses

Total labour cost for this employee during the period.

expenses = engagedHrs × hourlyCost

Revenue

Revenue attributed to this employee, prorated by their worklog contribution to priced requirements.

revenue = SUM(req_price × (my_worklog_seconds / total_req_worklog_seconds))

Net Profit

Revenue minus expenses for this employee.

profit = revenue − expenses

Employee Drawer

Detail KPIs GET /report/member/worklogs

Total Hours

Total available working hours (capacity) for the employee during the selected period.

totalWorkingHrs = periodCapacity
// periodCapacity = workingDaysInRange × dailyHours
// workingDaysInRange = non-holiday weekdays matching user's schedule in date range

Engaged

Total actual hours logged by this employee (from worklogs) during the period.

engagedHrs = SUM(worklog.time_in_seconds) / 3600
// Includes active timers: if end_datetime IS NULL, adds elapsed time up to NOW

Occupancy

How much of the employee's available capacity was utilized.

occupancy = ROUND((engagedHrs / totalWorkingHrs) × 100)%
Excellent — ≥ 90% Good — ≥ 75% Low — < 75%

Efficiency

Percentage of this employee's assigned tasks that were completed on or before their due dates.

efficiency = ROUND((completedOnTime / totalAssignedTasks) × 100)%

// completedOnTime = tasks WHERE status = "Completed"
// AND (end_date IS NULL OR completed_at ≤ end_date)
// If completed_at is null but task has deadline → NOT on time
// Returns 0 if totalAssignedTasks = 0
Excellent — ≥ 90% Good — ≥ 75% Low — < 75%

Work History Table

Worklog Entries

Individual time log entries from GET /report/member/worklogs. Each row shows:

date — worklog date (company timezone)
task — linked task name (clickable)
details — worklog description/notes
time — start_time – end_time (company timezone)
status — session status (e.g. "Completed")
duration — engaged time in HH:mm format

Helper Functions

Shared Logic

computeHourlyCost(profile)

Derives the hourly cost for an employee. Used in expense and rate calculations.

// Priority 1: Direct hourly rate
IF profile.hourly_rates → RETURN hourly_rates

// Priority 2: Derive from annual salary
annualWorkingHours = (daysPerWeek × 52 − publicHolidays − personalLeaves) × dailyHours
hourlyCost = salary_yearly / annualWorkingHours

getScheduleStats(workingHours)

Parses the working hours JSON config (company or user level) to determine schedule parameters.

// Modern format:
{ start_time: "09:00", end_time: "18:00", break_time: 60, working_days: ["monday", ...] }

dailyHours = (end_time − start_time − break_time) in hours
daysPerWeek = working_days.length

// Legacy format: { monday: { start_time, end_time }, ... }
daysPerWeek = count of active day keys
dailyHours = parsed from first active day

// Default (no config): 5 days/week, 8 hours/day

getLogDurationSeconds(log)

Calculates total elapsed seconds for a worklog entry. Handles active (running) timers.

seconds = log.time_in_seconds || 0
IF log.end_datetime IS NULL AND log.start_datetime EXISTS:
    seconds += (NOW − start_datetime) in seconds
RETURN seconds

computeUtilization(engagedHrs, profile)

Calculates the occupancy/load percentage for an employee during the selected period.

mergedSchedule = { ...companyWorkingHours, ...userWorkingHours }
periodCapacity = countWorkingDays(mergedSchedule, dateRange, holidays) × dailyHours
utilization = (engagedHrs / periodCapacity) × 100
// Returns 0 if periodCapacity = 0

User-level working hours override company defaults. Holiday dates and the date range determine the exact number of working days.

computeEfficiency(taskMemberRecords)

Calculates the on-time delivery rate for an employee's tasks.

totalTasks = taskMemberRecords.length
completedOnTime = COUNT WHERE:
    task.status = "Completed"
    AND (task.end_date IS NULL → always on time
         OR completed_at ≤ task.end_date)
    AND completed_at IS NOT NULL (if deadline exists)

efficiency = ROUND((completedOnTime / totalTasks) × 100)%
// Returns 0 if totalTasks = 0

computeRevenue(userLogs)

Prorates requirement revenue to individual employees based on their logged time contribution.

FOR EACH worklog in userLogs:
    reqPrice = requirement.quoted_price || requirement.budget || 0
    totalReqSeconds = total seconds logged by ALL users on this requirement
    revenue += reqPrice × (worklog_seconds / totalReqSeconds)

// If no one logged time on the requirement, revenue = 0
// In-house requirements with $0 price contribute $0 revenue
